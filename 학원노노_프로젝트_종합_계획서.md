## 1. 서비스 컨셉

### 한 줄 소개
> "호갱노노가 부동산 실거래가를 투명하게 만들었듯, 학원노노는 사교육 시장을 투명하게 만듭니다."

### 서비스명
**학원노노** (가칭)

### 핵심 가치
나이스 학원교습소 공공데이터를 지도 위에 시각화하여 세 이해관계자 모두에게 가치를 제공하는 플랫폼

| 이해관계자 | 제공 가치 |
|-----------|----------|
| 교사 | 학교 주변 사교육 현황 파악, 방과후 프로그램 기획 근거 |
| 학부모 | 지역별 학원 정보 탐색, 학원 리뷰/정보 공유 커뮤니티 |
| 학원 | 정보 등록/업데이트, 지역 타겟팅 홍보 채널 |

- PPT 또는 PDF 형태로 최대 **15매** (본문 기준, 부록 제외)
- 온라인(웹, 앱) 구축된 경우 해당 **URL 포함 제출**
- 교육 공공데이터 **1개 이상 필수 활용** (출처·라이선스 명시)
- AI 활용 과정 (프롬프트·모델·전처리·분석 절차) 요약 기재
- 결과물 내 팀명 외 학교명·개인명·지역명 등 **식별정보 기재 금지**
- **상업용 무료 폰트만** 사용

### 후속 지원
- 대상팀(일반): **범정부 공공데이터 AI 활용대회 통합본선 참여 추천**

- 공식 참가 증명서 및 수상 인증서(디지털 배지) 발급

## 5. 단계별 개발 계획

### 전체 로드맵

```
Phase 0 (1~2주)     Phase 1 (3~4주)         Phase 2 (대회 이후)     Phase 3 (장기)
─────────────────   ─────────────────────   ──────────────────     ──────────────
데이터 파이프라인    대회 출품용 MVP          커뮤니티 기능           플랫폼/수익화
├ API 수집           ├ 지도 시각화            ├ 카카오 로그인          ├ 학원 매니저

                     ├ AI 분석 리포트 ★
                     └ UI 마무리/배포
```

### Phase 0: 데이터 파이프라인 (1~2주)

**목표**: 전국 학원·학교 데이터를 수집하여 좌표와 함께 DB에 적재

| 순서 | 작업 | 세부 내용 | 산출물 |
|------|------|----------|--------|
| 0-1 | 프로젝트 초기화 | Next.js + Supabase + 패키지 설치 | 프로젝트 골격 |
| 0-2 | DB 스키마 설계 | academies, schools, district_stats, ai_reports 테이블 | SQL migration |
| 0-3 | 학원 데이터 수집 | 나이스 학원교습소 API 17개 시도교육청별 전수 수집 | academies_raw.jsonl |
| 0-4 | Geocoding | 카카오 REST API로 주소→좌표 변환 | academies_geocoded.jsonl |
| 0-5 | 학교 데이터 수집 | 나이스 학교기본정보 API 전수 수집 + Geocoding | schools_geocoded.jsonl |
| 0-6 | DB 적재 | Supabase 적재 + 행정구역별 집계 테이블 생성 | 운영 DB 완성 |

**나이스 API 호출 스펙**:
- 엔드포인트: `https://open.neis.go.kr/hub/acaInsTiInfo`
- 파라미터: `KEY`, `Type=json`, `pIndex`, `pSize=1000`, `ATPT_OFCDC_SC_CODE`
- 시도교육청 코드: B10(서울), C10(부산), D10(대구), E10(인천), F10(광주), G10(대전), H10(울산), I10(세종), J10(경기), K10(강원), L10(충북), M10(충남), N10(전북), O10(전남), P10(경북), Q10(경남), R10(제주)

**주요 필드**: 학원명(ACA_NM), 분야(REALM_SC_NM), 교습계열(LE_ORD_NM), 정원(PSNBY_THCC_CNTNT), 등록상태(REG_STTUS_NM), 도로명주소(FA_RDNMA), 수강료 관련 필드

### Phase 1: 대회 출품용 MVP (3~4주)

**목표**: 실제 동작하는 웹서비스 + 기획서 15매 제출

#### 1-1. 지도 기반 학원 분포 시각화

호갱노노의 UX를 참고하여:
- **줌아웃**: 동 단위 원형 마커 (학원 수 숫자 표시, 크기·색상으로 밀집도)
- **줌인**: 개별 학원 핀 마커 (클릭 시 인포윈도우)
- **분야별 색상 코딩**: 입시(파랑), 예능(보라), 체육(초록), 외국어(노랑) 등

#### 1-2. 분야별 필터

지도 위 좌측 패널에서 분야 토글:
- 전체 / 입시·검정 및 보습 / 국제화 / 예능 / 체육 / 외국어 / 직업기술 / 기타
- 선택한 분야만 지도에 표시
- 현재 지도 영역의 실시간 통계 표시

#### 1-3. 히트맵 뷰

마커 뷰와 히트맵 뷰를 토글로 전환:
- heatmap.js를 카카오맵 위에 오버레이
- 분야 필터와 연동
- "여기 학원이 많구나"를 직관적으로 전달

#### 1-4. 학교 주변 분석 대시보드

학교 검색 → 선택 시 사이드 패널:
- 반경 1/2/3km 설정
- 반경 내 분야별 학원 수 (도넛 차트)
- 시도 평균 대비 비교 (바 차트)
- **방과후 프로그램 제안**: 주변에 부족한 분야 자동 탐지
  - 예: "코딩/SW 학원이 2개로 적음 → 코딩 방과후 개설 제안"

#### 1-5. 지역 비교

시군구 2곳을 선택하여 나란히 비교:
- 총 학원 수, 분야별 비율, 평균 수강료
- 레이더 차트로 분야 구성 비교
- 자동 인사이트 텍스트 생성

#### 1-6. AI 분석 리포트 (★ 대회 핵심)

Claude API를 활용한 지역별 사교육 현황 분석:
- 지역 데이터(분야별 학원 수/비율, 수강료, 학교 수)를 프롬프트에 포함
- AI가 다음을 생성:
  1. **사교육 현황 요약** (3~4문장)
  2. **분야별 분석** (시도 평균 대비 비교)
  3. **학교 관점 제안** (방과후 프로그램 기획 3가지)
  4. **학부모 관점 정보**
  5. **데이터 한계 안내**
- 생성 결과를 DB에 캐싱 (24시간)

**AI 활용 과정 문서화** (대회 필수 요건):
- 사용 모델: Claude 3.5 Sonnet (또는 사용 가능 모델)
- 프롬프트 구조 및 전문
- 데이터 전처리 과정
- 출력 파싱 및 검증 방법

#### 1-7. 전국 현황 대시보드

기획서 캡처 및 발표용 인포그래픽 페이지:
- 전국 학원 총 수 (큰 숫자)
- 시도별 바 차트
- 분야별 도넛 차트
- 밀집도 TOP10 시군구

#### 1-8. 배포

- Vercel 배포 → 실제 접속 가능한 URL 확보
- 대회 기획서에 URL 포함

## 7. 대회 출품 범위 전략

### 반드시 구현 (심사 점수 직결)

| 기능 | 심사 기준 연결 |
|------|--------------|
| 지도 위 학원 분포 시각화 (분야별 필터 포함) | 활용가능성 ★★★ |
| AI 기반 지역 분석 리포트 생성 | AI활용도 ★★★ |
| 학교 주변 사교육 현황 분석 | 활용가능성 ★★★ |
| 실제 접속 가능한 웹서비스 URL | 독창성 + 활용가능성 |

### 기획서에만 담을 것 (로드맵 슬라이드)

| 기능 | 역할 |
|------|------|
| 학부모 리뷰/평점 커뮤니티 | "서비스 확장 가능성" 어필 |
| 지역 타래 게시판 | "호갱노노형 커뮤니티 모델" 어필 |
| 학원 매니저 대시보드 | "플랫폼 양면시장 모델" 어필 |
| 타겟팅 광고 수익모델 | "사업성/지속가능성" 어필 |

### 차별화 포인트

1. **호갱노노 비유**: 심사위원에게 서비스 컨셉을 즉시 이해시키는 한 줄
2. **현직 교사 경험**: "방과후 프로그램 기획에 이 데이터가 실제로 필요했다"는 진정성
3. **실제 동작 URL**: 대부분의 출품작이 기획서만 제출 → 동작하는 서비스는 강력한 차별화
4. **세 이해관계자**: 교사·학부모·학원 모두에게 가치를 제공하는 구조
5. **확장 로드맵**: 단순 공모작이 아닌 실제 서비스로의 발전 가능성

## 8. Phase 2: 커뮤니티 기능 (대회 이후)

### 사용자 인증
- 카카오 로그인 (Supabase Auth OAuth)
- 사용자 프로필: 닉네임, 관심 지역, 자녀 학교급

### 학원 리뷰/평점
- 학원별 상세 페이지
- 별점(1~5) + 텍스트 리뷰 + 태그(수업 좋음/시설 좋음/가격 적정 등)
- AI 리뷰 요약 (긍정/부정 키워드 추출)
- 신고 기능, 비속어 필터

### 지역 커뮤니티 (타래)
- 동 단위 게시판 (호갱노노 입주민 게시판 모델)
- 학원 추천/비추천 스레드
- 방과후 정보 공유
- 익명/실명 선택

### 알림 기능
- 관심 지역에 새 학원 등록/폐원 시 알림
- 관심 학원 수강료 변경 알림
